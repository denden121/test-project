# Сценарии тестирования: социальный вишлист

Пользовательские истории (user stories) и сценарии тестирования для веб-приложения «социальный вишлист». Сценарии оформлены **под написание e2e-тестов на Playwright**: у каждого кейса есть шаги с селекторами (`getByRole`, `getByLabel`, `locator('#id')`) и явные проверки (`expect(...).toHaveURL()`, `toBeVisible()` и т.д.).

**Как переносить в тесты:** см. [PLAYWRIGHT.md](./PLAYWRIGHT.md) — конвенции, основные селекторы, роуты и пример теста.

Сценарии разбиты по фичам:

| № | Фича | Файл |
|---|------|------|
| 1 | Регистрация и авторизация | [01-auth.md](./01-auth.md) |
| 2 | Создание и управление вишлистами | [02-wishlists.md](./02-wishlists.md) |
| 3 | Товары в вишлисте | [03-items.md](./03-items.md) |
| 4 | Шаринг и просмотр вишлиста | [04-sharing.md](./04-sharing.md) |
| 5 | Резервация подарков | [05-reservations.md](./05-reservations.md) |
| 6 | Скиды (вклады в подарок) | [06-contributions.md](./06-contributions.md) |
| 7 | Реалтайм-обновления | [07-realtime.md](./07-realtime.md) |
| 8 | Дополнительные сценарии (крайние случаи и UX) | [08-edge-cases.md](./08-edge-cases.md) |

| — | Конвенция и примеры для Playwright | [PLAYWRIGHT.md](./PLAYWRIGHT.md) |

## Приоритеты

| Приоритет | Группа сценариев |
|-----------|------------------|
| **Критично** | Резервация (5), скиды (6), реалтайм (7), приватность для владельца (5.2, 6.2) |
| **Высокий** | Создание/редактирование вишлистов и товаров (2, 3), шаринг и просмотр (4), просмотр/отмена своей резервации и вклада (5.3, 6.4) |
| **Средний** | Регистрация и авторизация (1), безопасность и доступ (8.3), переподключение WS (8.2) |
| **Низкий** | Крайние случаи (8.1), мобильный UX и копирование ссылок (8.4) |

Идентификаторы сценариев (T1.1.1, T5.2.1 и т.д.) удобны для отчётов о дефектах и трассировки к требованиям.
